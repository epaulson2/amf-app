## PDF Build Instructions

Follow these steps in order when assembling the final Sprint 1 PDFs.

1. Render all 11 HTML assets to PNG at 150 dpi using Playwright before beginning any PDF assembly. Render once per asset, save to a `/build/assets/sprint1/` staging directory. Do not re-render mid-assembly.

2. Apply the following size rules to every PNG before embedding:
   - **Half-page cards** — R1, O3, O4: render at 816 × 528 px (half of US Letter at 150 dpi, landscape orientation within a portrait page)
   - **Full-page diagrams** — R2, R3, R4, E3: render at 1275 × 1650 px (US Letter portrait at 150 dpi)
   - **Full-page landscape maps** — E1, E2: render at 1650 × 1275 px (US Letter landscape at 150 dpi)
   - **Full-page anchor/rhythm assets** — O1, O2: render at 1275 × 1650 px (US Letter portrait); when these appear as workbook fill-in pages, add a blank-template overlay before embedding (see step 6)

3. Build the **Textbook PDF** section by section following the TEXTBOOK table above. For assets that appear twice in the textbook (E1, E2, R4), embed the same PNG by reference — do not duplicate the file on disk. Use PDF object references so the file size stays lean.

4. Build the **Practice Plan PDF** following the PRACTICE PLAN table above. Place O4 first (inside front cover), then the Daily Session Template with O3 and O2 as persistent sidebar elements. Embed weekly session assets in the correct week/section blocks. Assets that recur across weeks (R1, E1, E2, O2) are embedded by reference — one PNG object, multiple placements.

5. Workbook pages that ARE assets (E1→Page 3, E2→Page 4, O1→Page 5, O2→Page 6) are embedded as full pages in the **Workbook PDF**. Do not scale them — use the full-page PNG dimensions from step 2 so they fill the page edge to edge.

6. For workbook fill-in pages (Pages 3, 4, 5, 6), apply a blank-template overlay after embedding the base asset PNG: remove all pre-filled text/labels from the asset using a white masking layer at PDF level. The outline structure (fretboard lines, keyboard keys, chord grid, rhythm positions) must remain visible; only the student-filled content areas should be blank. Maintain at minimum 12-point font size for any labels left visible.

7. For the R1 sidebar on Workbook Page 2, embed R1 as a half-page inset cropped to the right column. The ear log table occupies the left 65% of the page; R1 occupies the right 35%.

8. After assembling all three PDFs, run a cross-reference check: for every asset placement in the By Asset table above, confirm the asset appears in that document and section. Flag any missing placements before finalizing.

9. Export final PDFs to `/build/output/sprint1/`: `sprint1_textbook.pdf`, `sprint1_practice_plan.pdf`, `sprint1_workbook.pdf`.

10. Archive source PNGs to `/build/assets/sprint1/` with filenames matching asset IDs (e.g., `R1_focus_card_3_4.png`). These are the canonical render outputs — do not delete them after PDF assembly; they will be reused for web embeds and future sprint reference.
